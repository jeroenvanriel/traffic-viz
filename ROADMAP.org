** TODO bug: interpolation interferring with playback auto-starting when recording
When the user uses the "Record from beginning" functionality and the current step is not the first step, the state is first set to the first step, which causes some vehicles to "fly" to their starting positions, which is visible in the recording.
** TODO better timeline control
1. Currently, seeking through the timeline is only possible at "snapshots". When streaming, the "delta" packets are used to compute the state between snapshots. The main reason for doing so is to limit the amount of data traffic between backend and frontend. As a next step, we would like to allow slightly finer seeking as follows: the timeline slider does not "snap", but the closest snapshot is always shown. Only when the user stops seeking, by releasing the mouse button or after some fixed timeout, do we calculate the exact system state between snapshots by applying the necessary list of delta packets at once.

2. When te user clicks on a keyframe marker, we could optionally jump to the corresponding step.
